Two Book — VAT & Non-VAT POS

White Paper สำหรับร้านวัสดุก่อสร้าง (Odoo 19 · ประเทศไทย)

Module: two_book Version: 19.0.1.1.0 Depends: point_of_sale, account, stock, pos_merge_invoice

1. บทนำ

โมดูล Two Book ออกแบบสำหรับธุรกิจที่จดทะเบียนภาษีมูลค่าเพิ่ม (VAT)แล้ว แต่หน้าร้านไม่ได้ออกใบกำกับภาษีให้ลูกค้าทุกราย — เช่น ร้านวัสดุก่อสร้างที่ลูกค้าส่วนใหญ่เป็นช่าง ร้านค้าปลีก หรือผู้ซื้อมาขายต่อโดยไม่ขอใบกำกับ

ระบบแยกการขายเป็น 2 เล่มใน Odoo เดียวกัน:

เล่ม ประเภทบิล ใบกำกับภาษี ภาษีขาย (ภ.พ.30)
VAT Book ใบกำกับภาษี 7% ออกได้ / Merge เป็นใบเดียว นับเข้า Output VAT
Non-VAT Book บิลเงินสด ไม่ออก ไม่นับ (รายได้ภายใน)

2. ปัญหาที่แก้

  • Odoo POS มาตรฐานบันทึกทุกบิลผ่าน flow เดียว — ไม่แยก “กระเป๋า VAT” กับ “บิลเงินสด”
  • ร้านวัสดุขายจริงทุกบิล แต่ส่งสรรพากรเฉพาะยอดที่ออกใบกำกับ → สต็อกเล่มภาษีต้องแยกจากสต็อกหน้าลาน
  • ต้องสรุปยอด VAT / Non-VAT ต่อรอบขาย และเตรียมข้อมูล ภ.พ.30

4. การติดตั้ง

  1. ติดตั้งโมดูล two_book และ pos_merge_invoice (dependency)
  2. Upgrade บน server:
    -u two_book,pos_merge_invoice
  3. Restart Odoo และ hard-refresh หน้า POS (เคลียร์ cache assets)

5. การตั้งค่าครั้งแรก

ไปที่ Point of Sale → Configuration → [POS Shop] → แท็บ Two Book (VAT/Non-VAT)

การตั้งค่าคำอธิบาย
เปิดใช้งาน Two Bookเปิดระบบแยก VAT/Non-VAT สำหรับ POS นี้
ค่าเริ่มต้น: ออกใบกำกับภาษีแนะนำ ปิด — order ใหม่เป็น Non-VAT
Journal VAT / Non-VATแยกสมุดรายได้และภาษีขาย
Fiscal Position VAT / Non-VATNon-VAT ต้อง map ภาษี 7% → ไม่มีภาษี
คลัง WH/VAT, Virtual/VAT_Outมี default จาก module — ตรวจก่อน go-live
Operation Type (Tax Stock)Internal transfer — ว่างได้ (ระบบหาให้อัตโนมัติ)
รับของเข้าคลัง: สินค้าต้องมีใน WH/Stock (ขายได้ทุกบิล) และ WH/VAT (ขาย VAT ได้) — ถ้า WH/VAT ไม่พอ ระบบจะ log warning และบันทึก Stock Gap แทน

6. Flow ใช้งานหน้าร้าน (Counter)

  1. เปิด POS Session ตามปกติ
  2. เพิ่มสินค้าในตะกร้า
  3. ก่อนชำระเงิน — กดปุ่มบน Actionpad:
    • บิลเงินสด (Non-VAT) — ลูกค้าทั่วไป / ไม่ขอใบกำกับ
    • ใบกำกับภาษี (VAT) — ลูกค้าขอใบกำกับ / นิติบุคคล
  4. ชำระเงิน — ระบบตัดสต็อกและบันทึกบัญชีตามประเภทบิล
สำคัญ: เลือก VAT/Non-VAT ก่อน validate payment — หลังจ่ายแล้วแก้ประเภทบิลยาก

สิ่งที่เกิดขึ้นหลังชำระเงิน

ประเภท WH/Stock WH/VAT บัญชี
Non-VAT ตัด ไม่ตัด → Stock Gap รายได้ Non-VAT journal
VAT ตัด ตัด → Virtual/VAT_Out รายได้ + VAT 7%

7. Admin — Merge ใบกำกับ (pos_merge_invoice)

ใช้เมื่อลูกค้า VAT ซื้อหลายบิล แล้วต้องการใบกำกับใบเดียว

  1. ไป Point of Sale → Orders
  2. เลือก POS orders ที่เป็น VAT ของลูกค้าเดียวกัน
  3. Action: Merge and Create Invoice

ระบบจะ:

  • ปฏิเสธถ้ามี Non-VAT order ปน
  • ปฏิเสธถ้า order invoiced แล้ว
  • สร้าง Invoice ใบเดียวผ่าน VAT journal + fiscal position
  • บันทึก tax_invoice_number ให้ทุก order ที่ merge

หมายเหตุ: Merge เป็นเรื่องบัญชี — ไม่ตัดสต็อกซ้ำ (ตัดไปแล้วตอนขายแต่ละบิล)

8. รายงานและการตรวจสอบ

เมนูใช้ทำอะไร
Point of Sale → รายงาน Two Book Pivot/Graph แยกยอด VAT vs Non-VAT ต่อ session — เตรียม ภ.พ.30
Point of Sale → Stock Gap (Non-VAT) รายการสินค้าที่ขาย Non-VAT แต่ยังไม่ตัดเล่มภาษี (สถานะ Open)
POS Session (ฟอร์ม) สรุป two_book_vat_total, two_book_vat_tax_total, two_book_non_vat_total
POS Order (list/form) คอลัมน์ two_book_type, is_vat_order, tax_invoice_number

9. ปิดเดือน (บัญชี / สรรพากร)

ภ.พ.30 = ภาษีขาย (จากบิล VAT + merged invoices) − ภาษีซื้อ (จาก Vendor Bills มาตรฐาน Odoo) รายได้รวม (งบการเงิน / ภ.ง.ด.) = ยอดขาย VAT + ยอดขาย Non-VAT สต็อกเล่มภาษี → ดู Stock Gap report → reconcile WH/VAT ด้วย Inventory Adjustment หรือ Internal Transfer (กระจายเหตุผล: เบิกใช้, เสียหาย, ตัวอย่าง — ไม่ควรตรงกับยอด Non-VAT เป๊ะทุกเดือน)

11. Troubleshooting

อาการแนวทางแก้
ไม่เห็นปุ่ม VAT/Non-VAT บน POS เปิด enable_two_book ใน config · upgrade module · hard refresh POS
Non-VAT ยังมีภาษี 7% ตรวจ Fiscal Position Non-VAT ว่า map tax ถูกต้อง
ขาย VAT แล้ว WH/VAT ไม่ลด ตรวจ location config · สต็อก WH/VAT พอหรือไม่ · ดู server log
Merge ไม่ได้ ต้องเป็น VAT orders ทั้งหมด · ลูกค้าเดียวกัน · ยังไม่ invoiced
Stock Gap โตเรื่อยๆ ปกติสำหรับ Non-VAT — ต้อง reconcile เป็นระยะ (wizard ยังไม่มีใน v1)

12. Roadmap (ยังไม่มีใน v1)

  • Wizard reconcile Stock Gap แบบ batch (ตัด WH/VAT อัตโนมัติ)
  • Sync รับของเข้า WH/Stock → WH/VAT อัตโนมัติ
  • แยก template ใบเสร็จ POS สำหรับ VAT / Non-VAT
  • ACL จำกัดสิทธิ์สลับ VAT/Non-VAT (ถ้าต้องการ)
